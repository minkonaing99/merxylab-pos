Key design decisions:  Products are global (shared catalog across stores) but stock is per-store via StoreProduct.  OrderItem snapshots product_name at sale time so refunds/history stay accurate if product is renamed.  StockMovement is a single append-only ledger — stock_qty on StoreProduct is always derived from it.  UserStoreRole handles multi-store RBAC — one user can be cashier in Store A and manager in Store B.
